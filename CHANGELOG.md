# Changelog — Қазтілші

Все значимые изменения в проекте фиксируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).  
Проект придерживается [Семантического версионирования](https://semver.org/lang/ru/).

---

## [3.4.0] — 2026-03-31

### Добавлено
- **Smart Ensemble с критиком**: Gemini выступает как независимый критик — анализирует все варианты перевода и пишет рецензию (калька, сингармонизм, морфология). GPT-4o Ensemble получает и варианты, и критику для информированного синтеза. Критик non-blocking — если не отвечает, Ensemble работает как раньше
- **Лингвистическое исследование** (56KB, 966 строк): диалекты, история языка, региональные особенности, академические ресурсы (KazParC 371K предложений, Termincom 383K терминов, Apertium-kaz, KazRoBERTa). Сохранено в `docs/kazakh-linguistics-research.md`

---

## [3.3.0] — 2026-03-31

### Добавлено
- **Админ-панель** (`/#/admin`) с 5 страницами:
  - **Обзор**: статистика переводов, графики рейтинга движков и латентности (Recharts)
  - **Переводы**: полная таблица с поиском, фильтрами по языку/движку, пагинацией, удалением, экспортом CSV
  - **Движки**: карточки со статусом, вызовами, латентностью, успешностью, ошибками
  - **Лаборатория**: тестовый перевод с отображением всех движков бок о бок
  - **Настройки**: параметры самооценки, приоритет движков, версия, аптайм
- 6 новых API-эндпоинтов под `/api/admin/*`
- Адаптивный сайдбар с мобильным drawer

### Изменено
- Админ-панель полностью на русском языке
- Формат дат переключён на ru-RU

## [3.2.0] — 2026-03-31

### Добавлено
- Подключён Yandex Translate с реальным API-ключом (через SGEO) — движок теперь полноценно работает в production
- Деплой на домен `kmt.qdev.run` с полной SSL-защитой (Let's Encrypt) и обратным прокси Nginx
- Полная OpenGraph-разметка в `client/index.html`: `og:title`, `og:description`, `og:image`, `og:locale`, `og:locale:alternate`
- Twitter Card с `summary_large_image` и отдельным OG-изображением 1200×630px
- JSON-LD структурированные данные (`WebApplication` schema.org) для поисковых систем
- `robots.txt` и `sitemap.xml` в `client/public/`
- `manifest.json` для PWA-поддержки с иконками и фирменным цветом темы `#c89b37`
- Canonical URL (`https://kmt.qdev.run/`)

### Изменено
- Интерфейс приложения переключён **с казахского на русский язык** для лучшей доступности русскоязычным пользователям
- Удалены флаг-эмодзи из всех элементов интерфейса (кнопки выбора языка, заголовки движков)
- Описания движков и подписи кнопок переведены на русский

## [3.1.0] — 2026-03-31

### Добавлено
- **5 новых движков перевода:**
  - **Claude Sonnet** (Anthropic) — confidence 0.93, ключ `CLAUDE_API_KEY`
  - **DeepSeek Chat** — confidence 0.87, ключ `DEEPSEEK_API_KEY`
  - **Grok 3 Mini** (xAI) — confidence 0.85, ключ `GROK_API_KEY`
  - **Mistral Small** (Mistral AI) — confidence 0.84, ключ `MISTRAL_API_KEY`
  - **Perplexity Sonar** — confidence 0.83, ключ `PERPLEXITY_API_KEY`
- Соответствующие файлы движков: `server/engines/claude.ts`, `deepseek.ts`, `grok.ts`, `mistral.ts`, `perplexity.ts`
- Обновлён `ENGINE_LABELS` на фронтенде — все новые движки отображаются с правильными именами
- Обновлён `KEY_MAP` в `GET /api/engines` — корректное отображение переменных окружения для всех движков
- Обновлён `ENGINE_ENV_KEYS` в `server/engines/index.ts` для автоматической фильтрации движков без ключей

### Исправлено
- Исправлен баг с `latencyMs` в Ensemble-результате: при отсутствии ключей или пустом ответе поле теперь корректно содержит фактическое время выполнения вместо `undefined`
- Добавлена серверная валидация длины текста (максимум **5000 символов**) через Zod-схему в `routes.ts`
- Логи API-ответов на сервере теперь обрезаются до 200 символов JSON, предотвращая переполнение консоли при больших ответах
- Движки без API-ключей теперь **автоматически пропускаются при запуске** (логируется в консоль: `Skipping <engine>: <KEY> not set`), вместо возврата ошибки при каждом запросе перевода

## [3.0.0] — 2026-03-30

### Добавлено
- **Полный редизайн** — палитра, вдохновлённая казахской золотой степью: основной цвет `#c89b37`, тёплые бежевые и коричневые тона
- **Логотип** — стилизованный шаңырақ (навершие юрты), реализован как инлайн SVG-компонент `KaztilshiLogo`
- **Цикл самооценки** (`server/engines/self-eval.ts`):
  - GPT-4o оценивает финальный перевод по 5 критериям, каждый 0–2 балла (итого 1–10)
  - Критерии: точность смысла, грамматика, сингармонизм, естественность, порядок слов
  - Системный промпт самооценки на **казахском языке** (Сен — қазақ тілінің сарапшысысың)
  - Если оценка < 8 — GPT-4o предлагает улучшенный текст и оценивает повторно (до 2 итераций)
  - Результат включает `evalScore`, `evalIterations`, `evalIssues` в поле `meta` ответа API
- **Улучшенные правила казахского языка** (`server/engines/kazakh-rules.ts`):
  - Детализированные правила агглютинативной морфологии с примерами
  - Правила согласования согласных (оглушение/озвончение суффиксов)
  - Расширенный раздел антикалек с конкретными лексическими заменами (проблема→мәселе, информация→ақпарат и др.)
  - Раздел специфичных казахских конструкций (отрицание, вопрос, косвенная речь, принадлежность, бытие)
- **Антипаттерны** в Ensemble постредакторе — явный чеклист из 6 пунктов для GPT-4o
- **Few-shot примеры** в системных промптах (`KAZAKH_FEWSHOT_RU`, `KAZAKH_FEWSHOT_EN`):
  - 5 примеров для RU→KK с разбором типичных ошибок
  - 5 примеров для EN→KK
  - Примеры встроены во все AI-движки через `getSystemPrompt()`
- **Значок качества** (`QualityBadge`) в интерфейсе: отображает оценку самооценки (1–10) с цветовой индикацией:
  - Зелёный ≥ 9, жёлтый ≥ 7, красный < 7
- **Тёмная тема** с тёплыми коричневыми тонами (CSS-переменные в `index.css`)

### Изменено
- `postEditTranslation()` получил обновлённые few-shot примеры и расширенный чеклист антипаттернов
- Порядок приоритетов движков расширен: `ensemble → openai → claude → gemini → deepseek → grok → tilmash → mistral → perplexity → deepl → yandex`
- Системный промпт для Qwen 72B теперь написан **на казахском языке** для повышения качества ответов
- `translateWithAll()` возвращает `{ results, meta }` вместо просто `results`

## [2.0.0] — 2026-03-30

### Добавлено
- **Ensemble постредактирование** (`server/engines/postprocess.ts`):
  - GPT-4o сравнивает все успешные варианты и синтезирует наилучший перевод
  - Системный промпт на казахском с инструкциями для редактора
  - Результат размещается первым в списке вариантов
- **Разделённый интерфейс** в стиле Google Translate — два панели бок о бок
- **Голосовой ввод** через Web Speech API (`SpeechRecognition`) с кнопкой-микрофоном
- **TTS** — кнопка озвучивания перевода через `window.speechSynthesis`
- **Страница истории переводов** (`client/src/pages/history.tsx`):
  - Список последних 20 переводов из базы данных
  - Фильтрация по направлению перевода
  - Отображение использованного движка и оценки
- **Система оценок** (`POST /api/translations/:id/rate`):
  - Кнопки «Хорошо» / «Плохо» — пользователь может указать, насколько перевод понравился
  - Оценка сохраняется в SQLite-базе данных
- **Клавиатурное сокращение** `Ctrl+Enter` для запуска перевода
- Кнопка копирования перевода в буфер обмена с анимацией подтверждения

### Изменено
- Интерфейс переработан под двухпанельный layout
- Добавлен роутинг через Wouter (`/` — переводчик, `/history` — история)
- `GET /api/translations` — новый эндпоинт для получения истории
- `POST /api/translations/:id/rate` — новый эндпоинт оценки

## [1.0.0] — 2026-03-30

### Добавлено
- **Первый релиз** — базовый переводчик с русского и английского на казахский
- **4 движка перевода:**
  - GPT-4o (OpenAI)
  - Qwen 72B (HuggingFace / Tilmash)
  - DeepL API
  - Yandex Translate v2
- Параллельный запрос ко всем движкам через `Promise.allSettled()`
- Автоматический выбор лучшего результата по приоритету (`selectBest()`)
- Отображение всех вариантов перевода с именем движка
- **SQLite-хранилище** через `better-sqlite3` и Drizzle ORM
- Схема базы данных: таблица `translations` с исходным текстом, переводом, движком и временной меткой
- **Docker-сборка**: многоэтапный Dockerfile, `docker-compose.yml`, volume для персистентности данных
- **Nginx-конфиг** с SSL-терминацией и редиректом HTTP→HTTPS
- **`deploy.sh`** — скрипт автоматического деплоя на Ubuntu/Debian VPS
- Базовые правила казахской грамматики в системных промптах AI-движков
- Простой одностраничный интерфейс с выбором языка источника
